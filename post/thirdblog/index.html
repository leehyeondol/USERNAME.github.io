    • 깨달은점 : ..?

    • notion에작성한것.

    일단 늦음. 11시 시작

=============

테스트 코드 → ex/ 로그인이 안됐지만 로그인이 되어 토큰이 넘어왔다 가정하는경우.

- mokito 메소드 알아볼것. → 이미 토큰 받았다고 생각하고 코딩 할 수 있음.
- DB가 없어도 있는것처럼하여 코딩가능.

=============

- 자동화를 배우는중. pipelines → github와 연동.

- 쿠버네티스 (k8s, Kubernetes, 큐브, kube)는 **컨테이너화된 애플리케이션을 배포, 관리, 확장할 때 수반되는 다수의 수동 프로세스를 자동화하는 오픈소스 컨테이너 오케스트레이션 플랫폼**입니다.
- 리눅스 마스터 2급이상은 시험 보세요

- 자동화 → 보통은 **릴리즈 브랜치에** 걸어둠 : 릴리즈브렌치 = 버그 수정 후 빌드해보는 브랜치

- pipelines → pipelines → new pipelines → use the clasic editor → gihub → repository 선택 → 보통은 릴리즈 브랜치 선택 → task에서 이것저것 추가. → triggers에서 scheduled add해서 시간 변경 → continuous intergration에서  enable cintinuous intergration  클릭하여 브랜치 변경당 자동화되게 설정 → save

**⇒ci/cd 중 ci를 한거래**

- CI/CD (Continuous Integration/Continuous Delivery)는 **애플리케이션 개발 단계를 자동화하여 애플리케이션을 더욱 짧은 주기로 고객에게 제공하는 방법**입니다. CI/CD의 기본 개념은 지속적인 통합, 지속적인 서비스 제공, 지속적인 배포입니다.

- 버튜얼머신 → 톰켓에 올려서 → war파일 실행 → 장애가났을때 어딘지 알 수 없음.
- 쿠버네트스 → 장애가 났을때 알아서 죽이고 알아서 살려줘.
    
    ![Untitled](https://s3-us-west-2.amazonaws.com/secure.notion-static.com/3ed73b1b-00dc-4193-add7-ddaaf6e3d602/Untitled.png)
    
    - 버전 컨트롤 하는법
- **git의 remote란 무엇인가?** git에서의 remote 는 **외부의 깃헙 저장소가 있는 url에 대한 alias(별칭)을 만들어 관리하기 위한 명령어**이다. 이 명령어는 단순히 GitHub만을 위한 것이 아니며, GitLab, Garrit 과 같은 외부 저장소를 다루기 위해서 사용한다.

==============

- 아이디와 비번을 타이핑없이 자동화 시키는법.
- 공개키와 비공개키를 이용해야함.
- github → setting → 카테고리 deploy keys → add new 에서 해볼것. 하는법은 오늘 안알려줌. 자동화 실습때 알려줄꺼래. 공개되어있는 블로그 있는데, 넘어간대. 금요일 진행.
