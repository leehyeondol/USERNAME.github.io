    iaaS 를 이용한 openAI api 사용

    1. kafka 세팅(로컬)
    2. Producer 구현
    3. Azure VM에 kafka세팅
    4. App service에 배포
    5. Consumer 구현(python)

    1.1 docker 설정파일 작성 1.2 docker compose 실행 1.3 kafka 기본 명령

    • 깨달은점 : 도커가 있다. kafka도 있다. 메세지 브로커 역할? . docker compose 를 이용하기 위해서는 docker-compose.yml 파일을 작성해야한다. docker compose를 이용해서 도커에 kkafka를 설치한다. kafka 내에 topic을 생성한다. topic이 있어야 두 서버 연결가능. -> 컨슈머와 프로듀서 생성하여 연결.

    • notion에작성한것.

    deployment/kafka/docker-compose.yml 폴더와 파일을 생성한다.

docker compose로 도커 실행 → 카프카라는 이미지(하나의 서버)생성

docker compose ~~~ exec ~~~ 로 topic 생성

docker compose -f .yml경로 exec kafka bash로 배쉬 실행

배쉬에서 컨슈머 실행 → kafka-consle-consumer ~~~~

배쉬+컨슈머(프로듀서) 셋트 → 이거 두개면 서로 연결된거.

프로듀서는 그냥 배쉬하나 열어서 새로 만들면 그게 프로듀서임.

배쉬에서 그냥

docker compose -f .yml경로 exec kafka bash로 배쉬 실행

배쉬에서 컨슈머 실행 → kafka-consle-consumer ~~~~

이거 똑같이해주면 됨.

Kafka는 **서버 클러스터 내에서 데이터 스트림을 레코드로 유지하는 방식으로 작동하는 브로커 기반 솔루션**입니다. Kafka 서버는 여러 데이터 센터에 분산되어 있을 수 있으며 여러 서버 인스턴스에 걸쳐 레코드 스트림(메시지)을 토픽으로 저장하여 데이터 지속성을 제공할 수 있습니다.
